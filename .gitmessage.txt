# 🧾 Commit Message Template
#############################

# <타입>: <제목>
#  - 제목은 50자 이내
#  - 무엇을 변경했는지 명확하게 (왜? 보다는 무엇을)
#  - 마침표 금지
#  - 예) feat: 로그인 API 추가

# -------------------------------------
# 본문(선택 사항, 72자 기준 줄바꿈 권장)
#  - 변경 이유, 상세 내용, 추가 정보 등을 작성
#  - 여러 줄일 경우 "-"로 구분
#  - 예)
#    - JWT 기반 로그인 로직 추가
#    - 프로필 이미지 컬럼 추가
# -------------------------------------

# -------------------------------------
# Footer(선택 사항)
#  - 이슈 트래킹 번호 또는 부가 정보
#  - 예) Closes: #101
# -------------------------------------

#############################
# Commit Type Guide
#############################
# ✨ feat:     새로운 기능 추가
# 🐛 fix:      버그 수정
# ♻️ refactor: 기능 변화 없는 리팩토링
# 📝 docs:     문서 수정 (README 등)
# 💡 chore:    설정/오타/주석 등 중요도 낮은 변경
# ✅ test:     테스트 코드 추가/수정 (프로덕션 변경 없음)
# 🚚 build:    빌드/환경 관련 파일 변경
# 🚑 hotfix:   긴급 수정(프로덕션 버그)
#############################
